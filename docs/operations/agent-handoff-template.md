# Agent Handoff Template

The structured note the triage system compiles when escalating a ticket to a human agent or specialist team. The goal is that the receiving agent has everything they need before they open the ticket thread.

---

## Why this matters

A poor handoff costs time and CSAT. The receiving agent reads the thread from scratch, asks the player to repeat information they already provided, and takes longer to reach a resolution. The player notices.

A good handoff means the agent opens the ticket knowing: what happened, what has already been tried, what still needs to happen, and any flags that change how they should approach it.

---

## Template

```
AI TRIAGE HANDOFF
================

Ticket ID:        [ticket ID]
Date:             [date and time]
Handoff reason:   [why the AI escalated - e.g. legal threat, VIP churn, low confidence, repeat contact]

PLAYER
------
Player ID:        [player ID if provided]
VIP tier:         [0 / 1 / 2 / 3]
Contact count:    [how many times this player has contacted on this issue]
Prior resolution: [yes / no - was a resolution attempted previously]

ISSUE
-----
Classified as:    [intent type] (confidence: [score])
Tone detected:    [neutral / frustrated / angry / threatening / distressed]
Flags:            [list all active flags, e.g. legal_threat, repeat_contact, churn_risk]

What the player said (verbatim):
"[player's message]"

Known issue match: [yes / no]
If yes, issue ID: [ISSUE-YYYY-MM-DD-SHORT-DESCRIPTION]
Compensation eligible: [yes / no / pending verification]

INFORMATION ALREADY COLLECTED
------------------------------
[List any information the player already provided, e.g.:
- Transaction ID: TXN-884521
- Platform: iOS
- Purchase date: 14 April 2024]

INFORMATION STILL NEEDED
------------------------
[List what still needs to be collected before resolution is possible]

WHAT HAS ALREADY BEEN TRIED
-----------------------------
[List any Tier 1 steps already attempted and their outcome, or "none" if first contact]

RECOMMENDED NEXT ACTION
-----------------------
[What the triage engine suggests the agent should do first]

PRIORITY
--------
Score:   P[1-5]
SLA:     [time]
Reason:  [why this priority was assigned]

ROUTING
-------
Assigned to:  [team name]
Reason:       [why this team]

---
Generated by AI triage. Review before acting. Not a replacement for reading the ticket.
```

---

## Notes for agents

This note is a starting point, not a script. The AI classified based on what the player wrote. You may have additional context from the ticket thread or account history that changes the picture.

If the AI got the classification wrong, use the [escalation-misfire issue template](https://github.com/alexandros-alexakis/ai-customer-support-agent/issues/new/choose) to log it. That is how the system improves.

If the KB was missing something that would have let the AI resolve this without escalating, use the [KB gap issue template](https://github.com/alexandros-alexakis/ai-customer-support-agent/issues/new/choose).
