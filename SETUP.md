# Setup Guide

Full installation and configuration instructions.

---

## Prerequisites

| Requirement | Minimum version | Check command |
|---|---|---|
| Python | 3.10 | `python --version` |
| pip | 23.0 | `pip --version` |
| Git | Any | `git --version` |
| Anthropic API key | Required for multilingual and LLM features | console.anthropic.com |

---

## 1. Clone the repository

```bash
git clone https://github.com/alexandros-alexakis/customer-support-agent.git
cd customer-support-agent
```

---

## 2. Create a virtual environment

Always use a virtual environment. This keeps project dependencies isolated from your system Python.

```bash
# Create
python -m venv venv

# Activate - macOS / Linux
source venv/bin/activate

# Activate - Windows
venv\Scripts\activate
```

Your terminal prompt should now show `(venv)` at the start.

To deactivate when done:
```bash
deactivate
```

---

## 3. Install dependencies

```bash
pip install -r requirements.txt
```

This installs:

| Package | Version | Purpose |
|---|---|---|
| `anthropic` | >=0.25.0 | Claude API client |
| `chromadb` | >=0.4.0 | Local vector database for RAG |
| `sentence-transformers` | >=2.7.0 | Local embeddings for semantic search |
| `flask` | >=3.0.0 | Web server for Zendesk webhook |
| `gunicorn` | >=21.0.0 | Production WSGI server |
| `python-dotenv` | >=1.0.0 | Loads `.env` into environment |
| `requests` | >=2.31.0 | HTTP client for Zendesk API |
| `pytest` | >=8.0.0 | Test runner |

**First install note:** `sentence-transformers` downloads a ~80MB model on first use. This is normal. Subsequent runs use the cached model.

---

## 4. Configure environment variables

```bash
cp .env.example .env
```

Open `.env` and fill in your values. See `CONFIGURATION.md` for full documentation of each variable.

Minimum required for basic engine use:
```bash
ANTHROPIC_API_KEY=your_key_here
```

Additional variables required for Zendesk integration:
```bash
ZENDESK_SUBDOMAIN=yourcompany
ZENDESK_EMAIL=support@yourcompany.com
ZENDESK_API_TOKEN=your_token
WEBHOOK_SECRET=your_secret
```

---

## 5. Sync the knowledge base (RAG)

This loads all markdown files from `knowledge-base/` into ChromaDB for semantic search. Required before using the retriever.

```bash
python rag/kb_sync.py
```

Expected output:
```
Syncing knowledge base from: /path/to/knowledge-base
  Synced faq-payments.md: 5 chunks
  Synced faq-account-access.md: 6 chunks
  ...
Sync complete: 12 files, 67 chunks
```

This creates `rag/chroma_store/` locally. It is in `.gitignore` and will not be committed.

---

## 6. Validate the setup

Run the following checks in order:

```bash
# Check 1: Example run works
python example_run.py
# Expected: 4 ticket results printed with intent, priority, escalation

# Check 2: Unit tests pass
pytest tests/ -v
# Expected: all tests pass

# Check 3: RAG retrieval works (requires kb_sync first)
python rag/example_rag.py
# Expected: similarity scores printed for each query
```

If all three pass, the setup is complete.

## Platform notes

**Windows:** Use `venv\Scripts\activate` not `source venv/bin/activate`. All Python commands are otherwise identical.

**macOS with Apple Silicon (M1/M2/M3):** `sentence-transformers` may require Rosetta or specific torch builds on some configurations. If you see errors, try `pip install torch --index-url https://download.pytorch.org/whl/cpu` before installing requirements.

**Python version conflicts:** If `python` points to Python 2 on your system, use `python3` and `pip3` explicitly.
